# System Architecture

SmartCare follows a **three-tier architecture** consisting of a presentation layer, application layer, and data layer. This architecture separates the user interface, business logic, and data storage, improving maintainability, scalability, and system organization.

## Architecture Components

### 1. Presentation Layer (Frontend)

- Built using **Next.js** and **Tailwind CSS**
- Provides interfaces for Patients, Doctors, and Administrators
- Sends requests to the backend through RESTful APIs
- Displays data and system responses to users

### 2. Application Layer (Backend)

- Built using **Laravel (PHP)**
- Handles business logic and system operations
- Processes symptom analysis and specialty recommendations
- Manages authentication, appointments, doctors, and user data
- Communicates with the database

### 3. Data Layer (Database)

- Built using **MySQL**
- Stores patient information
- Stores doctor and specialty data
- Stores appointment records
- Ensures data consistency and integrity

## System Communication Flow

1. Patient, Doctor, or Administrator interacts with the frontend application.
2. The frontend sends requests to the Laravel REST API.
3. The backend processes the request and performs the required business logic.
4. The backend retrieves or updates data in the MySQL database.
5. The response is returned to the frontend and displayed to the user.

```
Patient / Doctor / Admin
            │
            ▼
     Next.js Frontend
            │
       RESTful API
            │
            ▼
      Laravel Backend
            │
            ▼
      MySQL Database
```
