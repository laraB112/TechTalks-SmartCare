# API Planning

## Overview

The SmartCare platform will use a RESTful API architecture to enable communication between the Next.js frontend and the Laravel backend. The API will handle user authentication, symptom analysis, doctor management, and appointment scheduling.

---

## Authentication Endpoints

| Method | Endpoint | Description |
| --- | --- | --- |
| POST | /api/register/ | Register a new patient account |
| POST | /api/login/ | Authenticate user and return access token |
| POST | /api/logout/ | Log out the authenticated user |
| GET | /api/profile/ | Retrieve user profile information |

---

## Symptom & Recommendation Endpoints

| Method | Endpoint | Description |
| --- | --- | --- |
| POST | /api/symptoms/ | Submit symptoms for analysis |
| GET | /api/recommendation/ | Retrieve recommended medical specialty |

---

## Doctor Endpoints

| Method | Endpoint | Description |
| --- | --- | --- |
| GET | /api/doctors/ | Retrieve all doctors |
| GET | /api/doctors/{id} | Retrieve doctor details |
| GET | /api/doctors/specialty/{specialty} | Retrieve doctors by specialty |
| GET | /api/doctors/{id}/availability | Retrieve available appointment slots |

---

## Appointment Endpoints

| Method | Endpoint | Description |
| --- | --- | --- |
| POST | /api/appointments | Create a new appointment |
| GET | /api/appointments | Retrieve user appointments |
| PUT | /api/appointments/{id} | Reschedule an appointment |
| DELETE | /api/appointments/{id} | Cancel an appointment |

---

## Administrator Endpoints

| Method | Endpoint | Description |
| --- | --- | --- |
| POST | /api/admin/doctors | Add a new doctor |
| PUT | /api/admin/doctors/{id} | Update doctor information |
| DELETE | /api/admin/doctors/{id} | Remove a doctor |
| GET | /api/admin/statistics | View system statistics |
| GET | /api/admin/appointments | View all appointments |

---

## API Communication Flow

```
Frontend (Next.js)
        │
        ▼
RESTful API Requests
        │
        ▼
Laravel Backend
        │
        ▼
MySQL Database
```
