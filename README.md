# TechTalks-SmartCare
# Overview:
SmartCare is a platform for a single hospital where patients write down their symptoms to get an automatic specialty recommendation, then book available slots. Doctors view daily patient schedules, and the Admin controls the doctor database and monitors patient assignments.
This eliminates patient guesswork and reduces receptionist workload.
# Users:
 # 1.patient:
- Enter symptoms
- Get recommended specialty
- Book appointments
- View booked appointments

 # 2.Doctors:
- view daily schedule
- See assigned patients

 # 3. Admin:
- Manage doctors(add/edit/delete)
- Assign doctors to specialties

# Technologies Used
 # frontend:
- Next.js
- javascript

 # Backend
- Laravel/php
- MySQL
  
# System Architecture
- REST API communication between frontend and backend

# project goal:
- simplify hospital appointment booking by matching patient symptoms with the correct medical specialty and reducing manual scheduling work.
